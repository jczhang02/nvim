# Neovim 0.12 Fresh Config — Design Spec

**Date**: 2026-05-04
**Author**: JC Zhang
**Status**: Approved (brainstorm complete)
**Source**: rebuild of `~/.config/nvim` (current: nvimdots-derived, ~70+ plugins)

## 1. Goals (combined drivers)

- **Modernize** — adopt Neovim 0.12 native APIs (`vim.lsp.config`, `vim.lsp.enable`, native `gc`/`gcc`, `vim.treesitter.start`, `vim.loader.enable`).
- **Declutter** — drop redundant plugins; consolidate via `snacks.nvim`; replace `nvim-cmp` stack with `blink.cmp`; replace `none-ls` stack with `conform.nvim` + `nvim-lint`.
- **Refactor structure** — flatten the legacy three-tier `core/modules/user` layout to Folke-style `lua/plugins/<name>.lua` (one plugin per file).
- **Performance-first** — startup target ≤ 25 ms (current ~26 ms on 0.11 rewrite). All non-essential plugins lazy-loaded.

## 2. Non-Goals

- AI assistance (Copilot, CodeCompanion, MCP-Hub) — explicitly removed.
- Telescope ecosystem — replaced by `snacks.picker`.
- Pure-native treesitter (no `nvim-treesitter`) — keeps `nvim-treesitter` `main` branch as the parser/query manager (the recommended 0.10+ posture).
- Mason removal — keep Mason for binary management; do not migrate to mise/system packages this round.

## 3. Scope: languages & workflows

Retain: LaTeX (vimtex), Markdown (render-markdown), C/C++ (clangd), Rust (rustaceanvim), Go (gopls), Python (ruff + zuban), Lua (lua_ls), TypeScript/JavaScript (ts_ls), Shell (bashls + shellcheck), JSON/HTML, BibTeX, CSV, ebuild (Gentoo).
Drop: AI tooling, qmake (re-add on demand), Go enhancement plugin (`go.nvim`).
Keep tooling: DAP full stack, im-select (CJK input switch), pangu (CJK whitespace), pastify (image paste).

### 4.4 Keymap strategy

- **Embedded**: plugin-related bindings live in each spec's `keys = {…}` field with `desc = "…"`. lazy.nvim uses these for keystroke-level lazy-loading; which-key v3 reads `desc` automatically.
- **Global**: window/buffer/tab/movement bindings live in `lua/config/keymaps.lua` using `vim.keymap.set(..., { desc = "…" })`.
- **Groups** registered in which-key spec: `<leader>f` (find/picker), `<leader>l` (lsp), `<leader>d` (dap), `<leader>g` (git), `<leader>b` (buffer), `<leader>w` (window), `<leader>t` (terminal/tab), `<leader>x` (trouble/quickfix), `<leader>s` (search/replace), `<leader>c` (code), `<leader>r` (refactor/rename), `<leader>n` (notify/scratch), `<leader>p` (pastify/persisted), `<leader>a` (asyncrun/asynctasks).
- **Constraint**: every `keys` entry and every `vim.keymap.set` must include `desc`. No exceptions.

## 5. Component design

### 5.1 LSP (`lua/plugins/lsp.lua`)

Driven by `vim.lsp.config()` + `vim.lsp.enable()` (0.12 native). `nvim-lspconfig` is used only as the source of `default_config` dictionaries; it does not call `setup` on each server. `mason-lspconfig` ensures binaries; `automatic_installation = true` but no auto-setup. `neoconf.setup()` runs before LSP enable so per-project `.neoconf.json` overrides apply.

`LspAttach` autocmd binds buffer-local keymaps (`gd`, `gr` via `Snacks.picker.lsp_references`, `K`, `<leader>rn`, `<leader>ca`, `[d`, `]d`), attaches `lsp_signature`, and conditionally enables `vim.lsp.inlay_hint` based on `config.settings.lsp_inlayhints`.

Servers from `config.settings.lsp_deps`: `bashls`, `clangd`, `gopls`, `html`, `jsonls`, `lua_ls`, `ruff`, `ts_ls`, `zuban`. (`ts_ls` is added to the new config — it was previously installed out-of-band but referenced by `server_formatting_block_list` in the legacy settings.) Capabilities are augmented via `require("blink.cmp").get_lsp_capabilities()` against `*` (wildcard).

`server_formatting_block_list` is honored by passing `lsp_format = "fallback"` to conform and filtering `vim.lsp.buf.format` clients in the LspAttach handler.

### 5.2 Completion (`lua/plugins/blink.lua`)

`blink.cmp` (`version = "*"` for pre-built Rust binary). Sources: `lsp`, `snippets`, `path`, `buffer`, `spell`, with `latex` per-filetype for `tex`/`bib`/`markdown` and `tmux` opt-in per filetype if needed. Source labels mirror old cmp menu (`[LSP]`, `[SNIP]`, `[PATH]`, `[BUF]`, `[SPELL]`, `[LTEX]`, `[TMUX]`).

UI parity with old nvim-cmp: custom `┌─┐│└─` border via `completion.menu.border` table; rounded border for documentation; `winhighlight` reusing `Pmenu`/`PmenuBorder`/`PmenuSel`/`CmpDoc`/`CmpDocBorder`; three-column draw `kind_icon | label+description | source_name`; `width.max = 80` + `ellipsis = true` for long labels; `ghost_text` enabled (hidden when menu visible); `preselect = false`, `auto_insert = true`; cmdline mode enabled (replaces `wilder.nvim`).

Keymap: `<C-p>`/`<C-n>` select, `<C-d>`/`<C-f>` scroll docs, `<C-w>` hide, `<Tab>`/`<S-Tab>` select+snippet jump, `<CR>` accept.

`signature.enabled = false` — `lsp_signature.nvim` (already attached on LspAttach) handles in-insert signature display, which is a feature blink does not yet replicate.

**Trade-offs**: Loses old `lsp_scores` custom comparator and `cmp-under-comparator` underscore demotion. Compensated by blink's Rust fuzzy matcher and per-source `score_offset`. If underscored items rank too high in practice, tune `score_offset` for `lsp` source or add a custom transform in `sources.providers.lsp.transform_items`.

### 5.3 Format & Lint (`lua/plugins/conform.lua`, `lua/plugins/nvim-lint.lua`)

**conform**: format-on-save via `BufWritePre` autocmd (managed by conform itself). Per-filetype formatters: `lua → stylua`, `python → ruff_format + ruff_fix`, `go → goimports + gofumpt`, `rust → rustfmt`, `c/cpp → clang_format`, `sh/bash/zsh → shfmt`, `js/ts/vue/html/css/json/yaml/md → prettier`, `tex → latexindent`, `bib → bibtex-tidy`, `xml → xmlformat`. `lsp_format = "fallback"` for filetypes without an explicit formatter. `format_disabled_dirs` and `formatter_block_list` from `config.settings` are checked in the `format_on_save` callback; the old `format_modifications_only` setting and `lsp-format-modifications.nvim` are dropped (conform's range format covers visual selection on demand).

**nvim-lint**: `vint` for vimscript, `shellcheck` for sh/bash, triggered on `BufReadPost`/`BufWritePost`/`InsertLeave`.

### 5.4 Snacks integration (`lua/plugins/snacks.lua`)

Modules enabled: `bigfile` (1.5 MB threshold), `bufdelete`, `dashboard`, `explorer` (`replace_netrw = true`), `indent` (with `scope`), `input`, `notifier` (3000 ms timeout), `picker`, `quickfile`, `scope`, `scratch`, `statuscolumn`, `terminal`, `words`.

Picker keymaps (all under `<leader>f` group): `ff` files, `fg` grep, `fb` buffers, `fr` recent, `fc` cmd history, `fh` help, `fs` lsp_symbols, `fw` lsp_workspace_symbols, `fd` diagnostics, `fu` undo, `fp` projects.
Other: `<leader>e` explorer, `<C-\>` terminal toggle (n+t modes), `<leader>bd` bufdelete, `<leader>.` scratch.

Loaded eagerly (`lazy = false, priority = 1000`) because `bigfile` and `quickfile` need to be active before any buffer reads.

### 5.5 Treesitter (`lua/plugins/treesitter.lua`)

`nvim-treesitter` on `main` branch (no implicit highlight; we register `vim.treesitter.start()` ourselves via FileType autocmd in `config/autocmds.lua`). `:TSUpdate` runs on plugin install/update. Parsers from `config.settings.treesitter_deps`: bash, c, cpp, css, go, gomod, html, javascript, json, latex, lua, make, markdown, markdown_inline, python, rust, typescript, vimdoc, vue, yaml.

Dependencies all loaded with treesitter:
- `nvim-treesitter-textobjects` (`main` branch) — selection/movement around functions, classes, parameters
- `nvim-treesitter-context` — sticky function header
- `vim-matchup` — `%` jumps, enhanced text objects
- `nvim-ts-autotag` — auto close/rename HTML/JSX tags
- `rainbow-delimiters.nvim` — colored brackets
- `nvim-ts-context-commentstring` — JSX-style nested comment strings (used by native `gc` via `vim.b.commentstring` hook)

`treehopper` is dropped; flash.nvim's treesitter mode covers selection-by-tree-node.

### 5.6 DAP (`lua/plugins/dap.lua`)

`nvim-dap` + `nvim-dap-ui` (with `nvim-nio`) + `mason-nvim-dap` (installs `codelldb`, `delve`, `debugpy`). DAP keymaps under `<leader>d`: `c` continue, `b` toggle breakpoint, `B` conditional breakpoint, `i` step into, `o` step over, `O` step out, `t` terminate, `r` repl, `u` ui toggle, `C` close ui (preserves your existing `<leader>dC` binding).

### 5.7 Performance

- `vim.loader.enable()` first line of `init.lua`.
- All plugins `lazy = true` except `catppuccin`, `snacks.nvim`, `nvim-treesitter`.
- Event triggers grouped per §6 in brainstorm transcript: `BufReadPre/BufNewFile`, `InsertEnter`, `CursorHold`, `LspAttach`, `BufWritePre`, `BufWritePost/InsertLeave`, `VeryLazy`, `cmd = {…}`, `ft = {…}`, `keys = {…}`.
- `mason` ensure-installed runs asynchronously (default).
- `snacks.bigfile.size = 1.5MB` disables treesitter/LSP/indent on large files.
- Startup target: ≤ 25 ms (validated via `nvim --startuptime`).

## 6. Migration plan

Nine-phase incremental migration, isolated via `NVIM_APPNAME=nvim-new` until phase 9. Total ~4 hours hands-on:

| Phase | Focus | Time |
|---|---|---|
| 0 | Backup, `0.12-rewrite` branch, clear plugin caches | 5 min |
| 1 | Skeleton + `config/*` + `colorscheme` + `snacks` + `treesitter` | 30 min |
| 2 | `lsp` + `blink` + `conform` + `nvim-lint` + `glance` + `tiny-inline-diagnostic` | 60 min |
| 3 | UI plugins (bufferline, lualine, dropbar, edgy, neoscroll, which-key, trouble, todo-comments) | 30 min |
| 4 | Editor plugins (flash, surround, autoclose, mini-align, hlslens, vim-cool, sleuth, grug-far, smart-splits) | 20 min |
| 5 | Git + DAP + tools (gitsigns, diffview, fugitive, dap, bqf, highlight-colors, pastify, persisted, suda, debugprint, asyncrun, im-select, pangu) | 30 min |
| 6 | Lang configs (rust, latex, markdown, python, csv, gentoo) | 30 min |
| 7 | Snippet migration (`luasnippets/` + `snips/` → `lua/snippets/`) | 15 min |
| 8 | Performance verification (`--startuptime`, `:Lazy profile`, `:checkhealth`) | 10 min |
| 9 | Switch (`stow` new tree, alias old as `nvim-old` for 1–2 weeks) | 5 min |

## 7. Risks & mitigations

| Risk | Mitigation |
|---|---|
| blink.cmp ranking differs from cmp's `lsp_scores`/`cmp-under-comparator` and harms day-to-day flow | Tune `score_offset` per source; add `transform_items` to demote underscored items if needed; revert to `nvim-cmp` is feasible since stack is isolated to one file |
| snacks.picker missing a telescope feature you used (e.g. `live_grep_args`) | snacks has matching pickers for most cases; for live-grep-args specifically, snacks supports `--` raw rg flags via input; if blocking, add `fzf-lua` as a single-plugin escape hatch |
| `vim.lsp.config()` requires Neovim 0.11+ — confirm Gentoo box is on 0.12 before phase 2 | `nvim --version` check in phase 0; if Gentoo emerge still blocked (per memory 264/266), build from source or use an AppImage fallback |
| `Comment.nvim` removal exposes a JSX/TSX commentstring gap | `nvim-ts-context-commentstring` already in tree; verify it hooks into native `gc` via `vim.b.commentstring` setting on `FileType` |
| LuaSnip path migration leaves orphaned snippets | Inventory `snips/snippets/` before phase 7; if VSCode JSON, point `from_vscode.lazy_load` at it; if custom, port to lua per-filetype |
| AI removal regrets | Branch-isolated; re-introducing CodeCompanion is a single-file add later |

## 8. Settings file (slimmed)

`lua/config/settings.lua` retains: `format_on_save`, `format_timeout`, `format_notify`, `formatter_block_list`, `format_disabled_dirs`, `diagnostics_virtual_lines`, `diagnostics_level`, `disabled_plugins`, `palette_overwrite`, `colorscheme`, `transparent_background`, `background`, `external_browser`, `lsp_inlayhints`, `lsp_deps`, `dap_deps`, `treesitter_deps`, `gui_config`, `neovide_config`, `dashboard_image`.

Removed: `use_copilot`, `use_chat`, `chat_lang`, `chat_api_key`, `chat_models`, `search_backend`, `format_modifications_only`, `null_ls_deps` (replaced by inlining formatter list in `conform.lua`), `server_formatting_block_list` (handled inline in lsp.lua's LspAttach).

## 9. Acceptance criteria

- `:checkhealth` passes for `lsp`, `treesitter`, `blink`, `snacks`, `conform`, `nvim-lint`, `dap`.
- Startup ≤ 25 ms on a typical session (`nvim --startuptime /tmp/start.log`).
- Open one file per major language (lua, py, rs, go, c, ts, tex, md) and verify: LSP attaches, diagnostics render via `tiny-inline-diagnostic`, completion shows source labels, format-on-save works, `gd`/`gr`/`K` work, treesitter highlight active.
- `<leader>ff` opens snacks picker; `<leader>e` opens snacks explorer; `<C-\>` opens snacks terminal.
- LaTeX file: `\ll` triggers `VimtexCompile`, completion includes LaTeX symbols, `pastify` inserts image with `\includegraphics`.
- DAP: set breakpoint in a Python or Rust file, `<leader>dc` runs to it, `dap-ui` opens.
- which-key shows all `<leader>X` group labels.
- No errors in `:messages` after a 1-minute editing session.

## 10. Open questions (deferred)

- Should `snacks.terminal` adopt named/multi-pane workflows comparable to `toggleterm`'s send-line/visual commands? (Defer until Phase 5 reveals friction.)
- Is `rainbow-delimiters` worth its draw cost in a perf-first build? (Keep for now; revisit if `:Lazy profile` shows it ≥ 5 ms.)
- LuaSnip vs `mini.snippets` — `mini.snippets` is leaner but less mature for LaTeX. Keep LuaSnip.
