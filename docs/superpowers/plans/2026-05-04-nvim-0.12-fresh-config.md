# Neovim 0.12 Fresh Config — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the current ~70-plugin nvimdots-derived Neovim config with a flatter, ~40-plugin Folke-style config that uses native 0.11+/0.12 APIs (`vim.lsp.config`, `vim.lsp.enable`, native `gc`), `blink.cmp` (Rust matcher), `conform.nvim` + `nvim-lint`, `snacks.nvim` consolidation, and per-plugin files under `lua/plugins/`.

**Architecture:** Folke-style flat layout (`lua/plugins/<name>.lua`, one plugin per file). `lua/config/{options,keymaps,autocmds,lazy,settings,icons}.lua` boots Neovim. lazy.nvim manages everything; all but `catppuccin`, `snacks.nvim`, and `nvim-treesitter` are lazy-loaded. AI tools (Copilot, CodeCompanion) and obsolete plugin stacks (`nvim-cmp`, `none-ls`, telescope, `nvim-tree`, `alpha`, `nvim-notify`, `toggleterm`, etc.) are removed.

**Tech Stack:** Neovim 0.11.7+ (0.12 preferred), lazy.nvim, blink.cmp, LuaSnip, nvim-lspconfig + mason + neoconf, conform.nvim, nvim-lint, nvim-treesitter (main branch), snacks.nvim, catppuccin (upstream), nvim-dap, vimtex, rustaceanvim, lualine, bufferline, gitsigns, which-key v3.

**Spec:** [docs/superpowers/specs/2026-05-04-nvim-0.12-fresh-config-design.md](../specs/2026-05-04-nvim-0.12-fresh-config-design.md)

**Note on Neovim version:** Current system has nvim 0.11.7 (Gentoo emerge for 0.12 was blocked per prior memory). All required APIs (`vim.lsp.config`, `vim.lsp.enable`, native `gc`, `vim.loader.enable`) are available in 0.11. Plan executes on 0.11.7; 0.12 adds polish but is not blocking.

**Strategy:** Build the new config under `NVIM_APPNAME=nvim-new` so the existing config keeps working until phase 9 cutover. Each task verifies via headless startup, `:checkhealth`, or manual smoke. Frequent commits on a `0.12-rewrite` branch.

```lua
-- ~/.config/nvim-new/lua/config/autocmds.lua
local aug = vim.api.nvim_create_augroup
local au  = vim.api.nvim_create_autocmd

au("TextYankPost", {
  group = aug("HighlightYank", { clear = true }),
  callback = function() vim.hl.on_yank({ higroup = "IncSearch", timeout = 200 }) end,
})

au({ "BufReadPost" }, {
  group = aug("LastPosition", { clear = true }),
  callback = function(args)
    local mark = vim.api.nvim_buf_get_mark(args.buf, '"')
    local lcount = vim.api.nvim_buf_line_count(args.buf)
    if mark[1] > 0 and mark[1] <= lcount then
      pcall(vim.api.nvim_win_set_cursor, 0, mark)
    end
  end,
})

au("BufWritePre", {
  group = aug("AutoMkdir", { clear = true }),
  callback = function(args)
    local dir = vim.fn.fnamemodify(args.file, ":p:h")
    if vim.fn.isdirectory(dir) == 0 then vim.fn.mkdir(dir, "p") end
  end,
})

au("FileType", {
  group = aug("CloseWithQ", { clear = true }),
  pattern = { "qf", "help", "man", "lspinfo", "checkhealth", "notify", "trouble", "DiffviewFiles" },
  callback = function(args)
    vim.bo[args.buf].buflisted = false
    vim.keymap.set("n", "q", "<cmd>close<CR>", { buffer = args.buf, silent = true })
  end,
})

-- enable native treesitter highlight where parser exists (nvim-treesitter main branch posture)
au("FileType", {
  group = aug("TSStart", { clear = true }),
  callback = function(args)
    local lang = vim.treesitter.language.get_lang(vim.bo[args.buf].filetype)
    if lang and pcall(vim.treesitter.language.add, lang) then
      pcall(vim.treesitter.start, args.buf, lang)
    end
  end,
})
```
